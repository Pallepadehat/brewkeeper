# Contributing to brewkeeper

Thanks for helping improve `brewkeeper`.

## Development setup

```bash
bun install
bun dev
```

## Pull request checklist

- Keep changes focused and easy to review.
- Run `bun run typecheck` before opening a PR.
- Update docs when behavior or UX changes.
- Add screenshots/GIFs for visible TUI changes when possible.

## Commit style

Use clear, imperative commit messages:
- `fix cask metadata parsing`
- `improve footer readability`
- `add release workflow`

## Reporting issues

Please include:
- OS + terminal emulator
- Bun and Homebrew versions
- clear reproduction steps
- expected vs actual behavior

## Code of conduct

By participating, you agree to follow `CODE_OF_CONDUCT.md`.
